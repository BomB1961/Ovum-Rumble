# Ovum Rumble

## C UI Flow 브랜치 정리

`feature/C-ui-flow` 브랜치는 C 담당 UI 작업의 초기 뼈대를 잡기 위한 브랜치입니다.

이 브랜치는 실제 기능 구현 완료 브랜치가 아닙니다. 메인 메뉴와 게임 UI가 어떤 흐름으로 구성될지 확인하기 위한 씬/화면 구조 초안입니다.

```
C-ui-flow
= UI 구조/뼈대만 잡은 브랜치

C-add-ui-features
= C-ui-flow 뼈대 + 실제 기능 추가한 브랜치

C-add-ui-features가 C-ui-flow 내용을 포함하고 있습니다.
```

## 작업 범위

- `00_MainMenu` 씬 UI 흐름 초안
- `01_Game` 씬 UI 흐름 초안
- 메인 메뉴 화면 배치 방향 확인
- 인게임 HUD 배치 방향 확인
- 결과 화면 배치 방향 확인
- 설정 화면 배치 방향 확인

## 구현 상태

| 항목 | 상태 | 설명 |
|---|---|---|
| 메인 메뉴 | 뼈대 구성 | 실제 기능 연결보다는 화면 흐름 확인용입니다. |
| 방 만들기 | 기능 구현 X | Mirror 네트워크 연동이 필요합니다. |
| 방 참가 | 기능 구현 X | Mirror 네트워크 연동이 필요합니다. |
| 인게임 HUD | 뼈대 구성 | 현재 턴, 알 개수, 시간 표시 영역을 잡아둔 상태입니다. |
| 결과 화면 | 뼈대 구성 | 결과 화면 배치와 버튼 위치를 확인하기 위한 상태입니다. |
| 설정 화면 | 뼈대 구성 | 설정 패널 흐름을 확인하기 위한 상태입니다. |
| 재시작 / 종료 버튼 | 기능 구현 X | 실제 동작 연결은 별도 구현 브랜치에서 처리합니다. |

## 기능 구현 여부

이 브랜치에서는 실제 기능 구현을 완료하지 않았습니다.

포함하지 않은 것:

- 실제 게임 시작 처리
- 실제 방 만들기 / 방 참가 처리
- Mirror 네트워크 동기화
- 실제 턴 변경 처리
- 실제 알 개수 연동
- 실제 승패 판단
- 결과 화면 자동 전환
- 버튼별 최종 동작 연결
- 최종 디자인 polish
- 고급 UI 애니메이션

## 디자인 관련

이 브랜치는 최종 디자인 작업 브랜치가 아닙니다.

UI 배치와 화면 흐름을 확인하기 위한 초안이며, 최종 비주얼 스타일링, 고급 UI 애니메이션, 세부 디자인 polish는 추가하지 않았습니다.

## 테스트 방법

이 브랜치는 기능 테스트용이 아니라 화면 흐름 확인용입니다.

1. Unity에서 프로젝트를 엽니다.
2. `00_MainMenu` 씬을 엽니다.
3. Play Mode에서 메인 메뉴 화면 구성이 보이는지 확인합니다.
4. `01_Game` 씬을 엽니다.
5. HUD, 결과 화면, 설정 화면의 배치 구조를 확인합니다.
6. 버튼 클릭 시 실제 기능이 동작하지 않을 수 있습니다.

## 이후 작업

실제 기능 구현은 `feature/C-add-ui-features` 브랜치에서 진행했습니다.

`feature/C-add-ui-features`에는 다음 항목이 포함됩니다.

- 메인 메뉴 버튼 연결
- 인게임 HUD 표시
- 결과 화면 버튼 처리
- 재시작 / 메인 메뉴 이동 / 종료 버튼 처리
- UI 테스트용 게임 시간 흐름
- README 테스트 방법 정리
