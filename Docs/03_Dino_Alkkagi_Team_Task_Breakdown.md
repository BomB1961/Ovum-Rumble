# 공룡 알까기 4인 역할 분담 및 일정표

문서 버전: v2.0  
작성 목적: 4인 팀원이 각자 무엇을 만들고, 어디까지 책임지는지 명확히 한다.

---

## 1. 팀 개발 기준

이 프로젝트는 4명이 동시에 개발하지만, 각자 만든 기능이 충돌하지 않도록 책임 영역을 나눈다.

가장 중요한 기준:

1. `main` 브랜치는 항상 실행 가능해야 한다.
2. 새 기능은 완성 전까지 별도 브랜치에서 작업한다.
3. Basic Core를 망가뜨리는 변경은 팀 테스트 후 병합한다.
4. 모든 외부 에셋은 사용 즉시 `Credits.md`에 기록한다.

---

## 3. Person A: 알 물리/입력 담당

### 3-1. MVP 필수 작업

담당 클래스:

- `EggController`
- `FlickInputController`
- `EggSpawner`

작업 내용:

- 알 Prefab 생성
- `Rigidbody` 설정
- `SphereCollider` 설정
- `Physic Material`로 마찰/탄성 튜닝
- 마우스 드래그 시작/종료 처리
- 발사 힘 계산
- 최소/최대 힘 제한
- 알 충돌 이벤트 발생

완료 기준:

- 알 하나를 클릭/드래그/발사할 수 있다.
- 알끼리 충돌하면 서로 밀린다.
- 힘이 너무 약하거나 너무 세지 않게 제한된다.
- Inspector에서 주요 수치를 조절할 수 있다.

### 3-2. 확장 작업

- 알 종류별 무게/탄성 차이
- HP 시스템
- 능력 알
- 알 외형 바리에이션

## 4. Person B: 룰/판정/빌드 담당

### 4-1. MVP 필수 작업

담당 클래스:

- `TurnController`
- `MotionResolver`
- `BoardFallZone`
- `WinConditionChecker`

작업 내용:

- 현재 턴 플레이어 관리
- 발사 후 입력 잠금
- 모든 알 정지 판정
- 최대 해석 시간 처리
- 낙하 Trigger 처리
- 생존 알 수 계산

완료 기준:

- 한 플레이어의 알이 0개가 되면 승패가 나온다.
- 매일 실행 가능한 빌드를 만들 수 있다.

### 4-2. 확장 작업

- 4인 턴 룰
- 제한 시간 룰
- 디버그 UI
- LAN 멀티 실험

> **디버그 UI**: 개발 중에 현재 상태, 알 속도, 생존 알 수 등을 확인하는 임시 UI다. 버그 추적에 유용하다.

## 8. 브랜치 전략

```text
main
  feature/A-egg-physics
  feature/B-rules
  feature/C-ui-flow
  feature/D-presentation
  feature/D-bomb-module
```

규칙:

- `main` 직접 수정 금지.
- 기능별 브랜치에서 작업.
- 병합 전 최소 1명이 실행 확인.
- 충돌이 나면 기능보다 Basic Core 보존을 우선한다.

## 9. 완료 기준(DoD)

### 9-1. 기능 완료 기준

각 기능은 아래를 만족해야 완료로 본다.

- Unity Editor에서 실행된다.
- Basic Core를 망가뜨리지 않는다.
- Inspector 연결 누락이 없다.
- 에러 로그가 없다.
- 담당자가 아닌 1명이 실행해 봤다.
- 변경 내용을 Git에 커밋했다.

### 9-2. 발표 빌드 완료 기준

- Windows 빌드가 실행된다.
- 2인 핫시트 한 판이 끝난다.
- 결과 화면이 나온다.
- 사운드 볼륨이 너무 크거나 작지 않다.
- 외부 에셋 출처가 기록되어 있다.
- 발표 당일 사용할 씬이 명확하다.

## 12. 에셋 사용 체크리스트

외부 에셋을 쓰기 전에 확인한다.

- 상업 사용 가능인가?
- 수정 가능인가?
- 저작자 표시가 필요한가?
- 원본 파일 재배포 금지 조건이 있는가?
- 유명 IP나 팬아트가 아닌가?
- `Credits.md`에 적었는가?

## 13. 팀 공통 금지사항

- Basic Core 완성 전 네트워크 작업을 main에 병합하지 않는다.
- 라이선스 불명 에셋을 쓰지 않는다.
- AI가 만든 코드를 이해하지 않고 붙이지 않는다.
- 새 기능 때문에 턴/승패가 망가지면 즉시 되돌리거나 비활성화한다.
- Day 12 이후 새 기능을 추가하지 않는다.
